# Goal

In this chat, I will give you the Use Case Sheet files that I've generated. Each Use Case Sheet is for an API endpoint that will exist in an API project. My goal is to have you ask me **clarifying questions**, **merge the sheets into one document**, and finally **restructure and expand** them into a downloadable YAML OpenAPI v3.1.0 Spec file.

## Output Workflow

1. If you feel like you are missing context, please ask me to fill the gap.
2. Ask me any clarifying questions that you may have on the Use Case sheets that I provide to you, try to keep it concise and in-scope
3. If you don't have any clarifying questions and don't feel like you're missing any context, generate the YAML OpenAPI v3.1.0 Spec File

## Output

- Prioritize **formatting**, **content quality**, and **convention consistency**
- If you are giving me the spec file, please give it to me in a **downloadable YAML file**
- If you are asking me clarifying questions, let's keep it in your typical output format

## Clarifying Questions Workflow

1. Come up with the most common scenarios or context implications in a multiple choice format *e.g.* A) Yes B) No, A) 50 B) 75 C) 100. Always include an *Other* options, unless it's a true or false question.
2. If the answer is in the multiple choice bracket, I will give you the letter to the answer. If it's not, I'll choose the *Other* option and type a response.

## Documents

- OpenAPI v3.1.0 Specification Documentation - <https://spec.openapis.org/oas/v3.1.0.html>

## Guardrails

- Do **not** invent anything on your own. My Use Case Sheets are your framework and my answers to your questions are a source of truth
